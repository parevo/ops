Why Ops

One host. One workspace.

Stop bouncing between Terminal, Docker Desktop, and random SSH tabs. Ops keeps fleet context, containers, and shells scoped to the server you’re on.

Native macOS

SwiftUI + SwiftData. No Electron shell — feels like a system app, with Keychain-backed credentials and hardened runtime.

Server-scoped

Active host lives in the sidebar. Metrics, Docker, files, tunnels, and terminals all follow the server you selected.

Operator speed

Command palette, multi-tab SSH with SwiftTerm, split panes, and alert actions that jump straight into htop or docker ps.

Quiet defaults

Threshold alerts, memory of what you ran, and Sparkle updates — without turning your machine into a second monitoring stack.

Workspace

Everything you’d open a shell for.

Built for people who live in production: inspect containers, watch fleets, tunnel ports, and keep a real PTY open.

SSH that stays alive

Multi-tab interactive shells over OpenSSH, optional password ASKPASS, split view, themes, and font size that actually apply to the live session — not a fake console.

# tab · edge-01 · shell
➜ ssh root@10.0.0.12
Welcome to Ubuntu 24.04 LTS
root@edge-01:~# htop
PTY held across tab switches — no reconnect spam.

Containers, images, volumes, networks, and Compose — inspected over SSH. Quick actions for exec, logs, and local port forwards tied to the active host.

Install

From DMG to dock in a minute.

Requires macOS 14+. Builds ship as universal DMG/ZIP from GitHub Releases, with Sparkle checking this site for updates.

Download the latest build

Grab the .dmg (or .zip) from Releases. The button above always points at the newest asset.

Open and install

Open the DMG and drag Ops onto the Applications shortcut — classic macOS install.

First launch on Gatekeeper

Ad-hoc signed builds may need right-click → Open the first time. After that, Sparkle can keep you current.

Add a server and go

Key or password auth via Keychain. Pick the active host in the sidebar — every module follows.
